Hrorm provides a mechanism for building {@link org.hrorm.Dao} objects.

Hrorm Dao objects support basic CRUD operations on a relational (SQL) database: insert, select, update, and delete.

Dao objects are created by using a {@link org.hrorm.DaoBuilder} or {@link org.hrorm.IndirectDaoBuilder}. These classes provide a simple, declarative, fluent interface for describing the relationship between the Java object to be persisted and the underlying schema that supports it.

While almost all the classes and methods in this package are public, there is little need to use them directly. Other than the DaoBuilder, Dao, HrormException, Where, and a few others most clients should not have to directly reference anything in this package.

Hrorm intentionally converts all checked SQLException instances that arise to HrormException which extends RuntimeException in its client interfaces. Of course, proper exception handling is essential to a sound persistence policy in an application, regardless of whether the particular exceptions are checked or unchecked.
